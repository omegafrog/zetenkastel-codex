# 03-Permanent Notes

## 목적
제텔카스텐의 핵심. 원자적(atomic)이고 독립적인 지식 단위를 영구적으로 저장합니다.

## 제텔카스텐 철학
> "한 개의 노트 = 한 개의 아이디어"

Permanent notes는 당신의 **외부 두뇌**입니다. 시간이 지나도 가치를 유지하며, 서로 연결되어 지식 네트워크를 형성합니다.

## 6가지 핵심 원칙

### 1. 원자성 (Atomicity)
- 하나의 노트는 하나의 아이디어만
- 분리 가능하고 재사용 가능
- "그리고"가 들어가면 나누기

### 2. 자기완결성 (Autonomy)
- 다른 노트 없이도 이해 가능
- 필요한 맥락 포함
- 6개월 후 봐도 이해할 수 있어야 함

### 3. 자기 언어 (Own Words)
- 완전히 자신의 언어로 작성
- 단순 인용/복사 금지
- 이해의 증거

### 4. 연결성 (Connectivity)
- 다른 노트들과 적극적으로 연결
- 고립된 노트는 가치 없음
- "이 아이디어는 어디와 연결되는가?"

### 5. 맥락 독립성 (Context Independence)
- 출처보다 개념에 집중
- "X가 말했다"보다 "Y이다"
- 다양한 맥락에서 재사용 가능

### 6. 영구성 (Permanence)
- 시간이 지나도 가치 유지
- 트렌드가 아닌 원리
- 지식의 복리 효과

## 작성 방법

### 단계별 프로세스
```
1. 소스 선택 (fleeting note, literature note, 프로젝트 경험)
    ↓
2. 핵심 아이디어 추출 (하나만!)
    ↓
3. 자신의 언어로 설명
    ↓
4. 관련 노트 연결
    ↓
5. 적절한 위치에 저장 (MOC 업데이트)
```

### 구조 권장사항
```markdown
# [개념/아이디어 제목]

## 핵심 아이디어
[한 문단으로 핵심 설명]

## 상세 설명
[필요한 경우 부연 설명, 예시]

## 연결
- [[관련-permanent-note]] - 관계 설명
- [[다른-permanent-note]] - 어떻게 연결되는지

## 출처
- [[02-literature-notes/원본]]
- [[04-projects/프로젝트명]]
```

## 템플릿 사용
`09-templates/template-permanent-note.md`

## 네이밍 전략

### 옵션 1: 고유 ID + 제목
- 형식: `YYYYMMDDHHMMSS-제목.md`
- 예: `20251010143000-의존성주입.md`
- 장점: 절대 중복 없음, 생성 시점 추적
- 단점: 읽기 어려움

### 옵션 2: 의미 있는 제목
- 형식: `개념명.md` 또는 `개념명-설명.md`
- 예: `의존성주입.md`, `의존성주입-장단점.md`
- 장점: 읽기 쉬움, 검색 용이
- 단점: 중복 가능성

### 권장: 하이브리드
- 주 제목: 의미 있는 이름
- aliases 활용으로 다양한 접근

## 품질 체크리스트

작성 후 자문:
- [ ] 하나의 아이디어만 담고 있는가?
- [ ] 출처 없이 이해 가능한가?
- [ ] 완전히 내 언어로 작성했는가?
- [ ] 최소 2개 이상의 노트와 연결되는가?
- [ ] 6개월 후에도 이해 가능한가?
- [ ] 다른 맥락에서도 유용한가?

## 연결 패턴

### 1. 계층적 연결
```
[[상위개념]]
    ↓
[현재 노트]
    ↓
[[하위개념]]
```

### 2. 순차적 연결
```
[[선행개념]] → [현재 노트] → [[후속개념]]
```

### 3. 주제별 연결
```
[현재 노트] ←→ [[같은주제-다른관점]]
              ←→ [[관련주제]]
              ←→ [[반대개념]]
```

### 4. MOC 연결
```
[[07-maps-of-content/MOC-주제명]]
    ↓
[현재 노트]
```

## 성장 전략

### 초기 (0-100개)
- 품질보다 양
- 다양한 주제 탐색
- 연결 패턴 학습

### 중기 (100-500개)
- 연결 강화
- MOC 구축
- 구조 노트 생성

### 후기 (500+개)
- 지식 네트워크 활용
- 창발적 인사이트 발견
- 글쓰기/프로젝트에 활용

## 흔한 실수

❌ **실수 1**: 너무 긴 노트
- 해결: 하나의 아이디어가 맞는지 확인, 필요시 분리

❌ **실수 2**: 단순 복사 붙여넣기
- 해결: 반드시 자신의 언어로 재작성

❌ **실수 3**: 연결 없는 고립된 노트
- 해결: 최소 2-3개 노트와 연결 필수

❌ **실수 4**: 맥락 의존적 작성
- 해결: 출처 언급 최소화, 개념 자체에 집중

❌ **실수 5**: 완벽주의
- 해결: 80% 완성도로 시작, 나중에 개선

## 활용 방법

### 글쓰기
1. 주제 선택
2. 관련 permanent notes 수집
3. MOC나 구조 노트로 정리
4. 노트들을 연결하여 글 작성

### 프로젝트
1. 프로젝트 관련 permanent notes 검색
2. 적용 가능한 지식 확인
3. 프로젝트에 활용
4. 새로운 인사이트를 다시 permanent note로

### 학습
1. 새 개념 학습
2. Permanent note로 정리
3. 기존 지식과 연결
4. 지식 네트워크 확장

## 성공 지표
- 주당 3-5개의 새 permanent notes
- 각 노트가 평균 3-5개 연결
- 월 1회 기존 노트 검토 및 개선
- 분기 1회 MOC 재구성

---
**핵심**: 지식의 원자 단위, 연결이 힘이다
