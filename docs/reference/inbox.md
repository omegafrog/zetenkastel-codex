# 00-Inbox

## 목적
처리되지 않은 모든 새로운 노트가 처음 도착하는 곳입니다.

## 사용 방법
- 빠르게 캡처한 정보, 아이디어, 링크 등을 임시로 저장
- 정기적으로 검토하여 적절한 폴더로 이동
- **이 폴더는 항상 비워두는 것을 목표로 함**

## 처리 흐름
```
Inbox → 검토 → 분류 → 적절한 폴더로 이동
```

1. 새로운 내용을 inbox에 저장
2. 주기적으로 검토 (일일/주간)
3. 각 노트를 분류:
   - 임시 메모 → `01-fleeting-notes/`
   - 외부 자료 요약 → `02-literature-notes/`
   - 정제된 지식 → `03-permanent-notes/`
   - 프로젝트 관련 → `04-projects/`
   - 영역 관련 → `05-areas/`
   - 삭제 또는 보관

## 템플릿 사용
새 노트 작성 시: `09-templates/template-inbox.md` 사용

## 팁
- Inbox가 쌓이지 않도록 주의
- 처리 시간을 정기적으로 확보 (예: 매일 오전/저녁 10분)
- GTD의 "2분 규칙": 2분 안에 처리 가능하면 즉시 분류
- 일주일 이상 방치된 노트는 우선 처리

## 정기 검토 체크리스트

- [ ] Inbox 비우기
