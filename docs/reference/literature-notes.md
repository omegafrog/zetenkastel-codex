# 02-Literature Notes

## 목적
외부 자료(책, 논문, 강의, 아티클 등)를 읽고 요약·정리한 노트입니다.

## 제텔카스텐에서의 역할
- 외부 지식의 진입점
- 원저자의 아이디어를 자신의 언어로 번역
- Permanent notes의 중요한 원천

## 특징
- **출처 중심**: 특정 외부 자료에 대한 요약
- **객관성**: 원저자의 주장을 정확하게 표현
- **자기 언어**: 단순 복사가 아닌 재해석
- **선택적**: 모든 내용이 아닌 중요한 부분만

## Literature Notes vs Permanent Notes

| Literature Notes | Permanent Notes |
|-----------------|-----------------|
| 외부 출처 의존 | 출처 독립적 |
| 저자 중심 | 개념 중심 |
| "X는 Y라고 말함" | "Y이다" |
| 맥락 포함 | 원자적 |

## 작성 방법

### 1. 메타데이터 기록
```markdown
**저자**: [이름]
**출처**: [[08-references/원본파일]]
**타입**: 책/논문/아티클/강의
**읽은 날짜**: YYYY-MM-DD
```

### 2. 핵심 내용 요약
- 자신의 언어로 재해석
- 중요한 개념, 논증, 데이터
- 저자의 주장과 근거

### 3. 인용구 선별
- 원문 그대로 보존할 가치가 있는 것
- 따옴표나 인용 블록 사용

### 4. 자신의 생각 구분
- "## 내 생각" 섹션으로 명확히 분리
- 비판, 질문, 연결 아이디어
- 이 부분이 permanent notes의 씨앗

## 템플릿 사용
`09-templates/template-literature-note.md`

## 처리 흐름
```
외부 자료 읽기
    ↓
Literature Note 작성
    ↓
가치 있는 인사이트 발견
    ↓
Permanent Note로 추출 (03-permanent-notes/)
    ↓
원본 파일은 08-references/에 보관
```

## 네이밍 규칙
- **방법 1**: `저자-책제목.md`
  - 예: `마틴파울러-리팩토링.md`
- **방법 2**: `책제목-저자.md`
  - 예: `리팩토링-마틴파울러.md`
- **아티클**: `제목-출처.md`
  - 예: `REST API 설계-InfoQ.md`

## 작성 팁
### ✅ 좋은 예
```markdown
## 주요 내용
저자는 단일 책임 원칙(SRP)을 "클래스는 하나의 변경 이유만
가져야 한다"고 정의한다. 이는 클래스의 응집도를 높이고
결합도를 낮춰 유지보수성을 향상시킨다.

## 내 생각
이는 [[03-permanent-notes/응집도와결합도]]와 연결된다.
현재 프로젝트 [[04-projects/리팩토링]]에서 UserService가
SRP를 위반하고 있음.
```

### ❌ 나쁜 예
```markdown
## 주요 내용
단일 책임 원칙은 중요하다. (너무 추상적)

혹은

[책 내용을 그대로 복사 붙여넣기] (자신의 언어로 재해석 안함)
```

## 다음 단계
Literature note를 작성한 후:
1. 가치 있는 개념을 식별
2. 각 개념을 permanent note로 추출
3. 여러 literature notes에서 나온 개념들을 연결
4. MOC 업데이트
