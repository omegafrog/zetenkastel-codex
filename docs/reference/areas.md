
# 05-Areas (PARA 통합)

## 목적
지속적으로 관리해야 하는 책임 영역과 관심사를 관리합니다.

## PARA의 Areas
- 끝이 없는 지속적 활동
- 일정 수준을 유지해야 하는 영역
- 장기적 역량과 관심사

## 제텔카스텐과의 통합
- Areas는 **주제별 지식 관리**의 컨테이너 역할
- 각 Area는 관련된 permanent notes들을 참조
- MOC(`07-maps-of-content/`)로 Area의 지식 지도 생성

## Area 예시
- 건강 관리
- 백엔드 개발 역량
- 팀 리더십
- 재무 관리
- 글쓰기
- 지속적 학습 주제 (예: 데이터베이스 최적화)

## Area 노트 구조
```markdown
# Area: [영역명]

## 목적
[이 영역을 관리하는 이유와 목표 수준]

## 현재 프로젝트
- [[04-projects/프로젝트1]] - 진행중
- [[04-projects/프로젝트2]] - 계획중

## 핵심 지식
- [[03-permanent-notes/개념1]]
- [[03-permanent-notes/개념2]]
- [[07-maps-of-content/MOC-주제]] - 지식 지도

## 주요 리소스
- [[02-literature-notes/책1]]
- [[references/자료1]]

## 정기 검토
마지막 검토: YYYY-MM-DD
다음 검토: YYYY-MM-DD

## 개선 목표
- 이번 분기 목표
- 장기 방향성
```

## Areas vs Projects 구분
| Areas | Projects |
|-------|----------|
| 지속적 관리 | 마감 있음 |
| "백엔드 역량" | "Spring 학습 완료" |
| "건강" | "10kg 감량" |
| 유지 목표 | 달성 목표 |

## 관리 방법
- 분기별/월별 정기 검토
- 새 프로젝트는 Area에서 파생
- Area 관련 permanent notes 계속 축적
